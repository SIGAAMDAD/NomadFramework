Detailed Description

Lock-free queue with object pooling to reduce GC pressure. This is a C# adaptation of boost::lockfree::queue.

WARNING: Node reuse via pooling makes this implementation susceptible to the ABA problem. In practice, with .NET's memory model and typical usage, the risk is low, but it exists. For mission‑critical scenarios, consider using System.Collections.Concurrent.ConcurrentQueue<T> or a versioned approach (e.g., packing pointer+version into a long).
